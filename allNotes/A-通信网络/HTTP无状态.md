##token
    Token 无状态的表现主要体现在服务器端不需要存储任何会话信息。在传统的基于会话的身份验证机制中，服务器需要在服务器端存储会话信息，包括会话标识符、用户信息、权限等。而在 Token 身份验证机制中，服务器不需要存储任何会话信息，只需要验证 Token 的有效性即可。

具体来说，Token 无状态的表现在以下几个方面：

//无需在服务器端存储会话信息：传统的基于会话的身份验证机制需要在服务器端存储会话信息，通常是在服务器的内存或数据库中。而在 Token 身份验证中，服务器不需要存储任何会话信息，只需要验证 Token 的有效性即可。这减轻了服务器的负担，使服务器更加轻量级和可扩展。

//无需在客户端和服务器之间进行状态维护：在基于会话的身份验证中，客户端和服务器之间需要进行状态维护，通常是通过 Cookie 或 URL 参数来传递会话标识符。而在 Token 身份验证中，Token 被包含在请求的头部或请求体中，客户端和服务器之间不需要进行状态维护。这简化了应用程序的架构和开发，并减少了网络开销。

//可扩展性和分布式支持：由于服务器不需要存储会话信息，Token 身份验证机制更适合于分布式和可扩展的应用程序。多个服务器可以独立验证 Token 的有效性，而无需共享会话信息。这使得应用程序可以更容易地进行水平扩展和部署在多个服务器上。

//总之，Token 无状态的表现在于服务器不需要存储任何会话信息，只需要验证 Token 的有效性。这带来了服务器的轻量级、可扩展性和简化了应用程序的架构和开发。
