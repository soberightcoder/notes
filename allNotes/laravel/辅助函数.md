``````php
PHP中的符号表是指变量符号表，它是一种数据结构，用于存储变量名及其对应的值。符号表在PHP的内部实现中起着重要的作用，它有以下几种主要的类型和作用：

//全局符号表（Global Symbol Table）： 全局符号表存储了全局作用域中定义的变量和它们的值。这包括了在全局作用域中定义的变量、常量和超全局变量（如$_GET、$_POST等）。全局符号表在整个脚本执行期间都是可用的。

//局部符号表（Local Symbol Table）： 局部符号表存储了函数或方法内部定义的变量和它们的值。每当函数或方法被调用时，都会创建一个新的局部符号表，用于存储该函数或方法内部的变量。当函数或方法执行完毕后，局部符号表会被销毁。

//类符号表（Class Symbol Table）： 类符号表存储了类中定义的属性和方法。当类被实例化时，会为每个实例创建一个类符号表，用于存储该实例的属性值。

这些符号表在PHP的内部实现中起着关键的作用，它们帮助PHP引擎管理变量的作用域、生命周期和访问权限。理解符号表的概念有助于开发人员更好地理解PHP变量的存储和访问方式，以及作用域的影响。
``````

```````php
/**
 * 辅助函数可以在任何地方都可以用的原理
 * 在对象里面可以直接引用 函数  函数在函数的符号表里面可以在任何的地方调用，当然要在同一个文件内；不然要引入文件；
 */

 function main1() {
    echo "main assistant";
 }

 class Assist
 {
    public function main(){
     //辅助函数；
        main1();
    }
 }
 $obj =new Assist;
 $obj->main();

// app()辅助函数的实现；
//helper.php 辅助函数
//verdor/laravel/framework/src/Illuminate/Foundation/helper.php
if (! function_exists('app')) {
    /**
     * Get the available container instance.
     *
     * @param  string|null  $abstract
     * @param  array  $parameters
     * @return mixed|\Illuminate\Contracts\Foundation\Application
     */
    function app($abstract = null, array $parameters = [])
    {
        if (is_null($abstract)) {
            return Container::getInstance();
        }

        return Container::getInstance()->make($abstract, $parameters);
    }
}
```````
